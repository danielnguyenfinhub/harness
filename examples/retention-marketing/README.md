# Example Harness — Mortgage Client Retention & Marketing

A complete harness generated by the `harness` meta-skill for the **mortgage-broker client retention & marketing** domain. Use it as a reference for what a finished harness looks like, or drop it into a broker project to run.

## What it does

Takes a broker from *"who should I contact?"* to an **approved, ready-to-send campaign package** — scanning the loan book for retention triggers, building consent-filtered audiences, writing compliant copy, and getting compliance sign-off. It **never auto-sends**: the harness stops at an approved plan the broker executes.

## Architecture

**Mode:** Agent Team — **Pipeline + Producer-Reviewer**

```
retention-analyst → audience-segmenter → campaign-copywriter ⇄ compliance-reviewer
   (opportunities)     (consent-filtered      (SMS/email copy)   (AU credit-law gate)
                         audiences)
                                    ↓
                  retention-orchestrator compiles the package
                              → AWAIT BROKER APPROVAL → broker sends
```

| Component | Type | Responsibility |
|-----------|------|----------------|
| `retention-analyst` | agent | Scan the loan book, prioritise opportunities (fixed-rate/IO/pre-approval expiry, anniversaries, rate review, equity release, win-backs, birthdays) |
| `audience-segmenter` | agent | Build clean, de-duplicated, **consent-filtered** audiences; assign SMS vs email |
| `campaign-copywriter` | agent | Write personalised, compliant SMS/email copy with one clear CTA |
| `compliance-reviewer` | agent | Gate copy + audience against AU credit-marketing law |
| `retention-orchestrator` | skill | Coordinate the team; compile the package; enforce the no-auto-send safety gate |
| `retention-opportunity-scan` | skill | Opportunity taxonomy + CRM tool mapping + prioritisation rubric |
| `audience-segmentation` | skill | Consent hard-gate → de-dupe → channel assignment pipeline |
| `campaign-copywriting` | skill | Channel-correct message craft + self-check |
| `mortgage-marketing-compliance` | skill | NCCP, RG 234, comparison rate, Spam Act, Privacy Act, DNC checklist (+ rules reference) |

## How to use it in a broker project

1. Copy `agents/` and `skills/` into the project's `.claude/` directory:
   ```bash
   cp -r examples/retention-marketing/agents  /path/to/project/.claude/agents
   cp -r examples/retention-marketing/skills  /path/to/project/.claude/skills
   ```
2. Add the harness pointer to the project's `CLAUDE.md` (see the snippet below).
3. Ensure the broker's CRM MCP server is connected (the analyst/segmenter call its list-builder, do-not-mail, and campaign tools).
4. Ask, e.g.: *"Build a fixed-rate-expiry retention campaign for the next quarter, SMS."* The `retention-orchestrator` skill takes over.

### CLAUDE.md pointer snippet

```markdown
## Harness: Mortgage Client Retention & Marketing

**Goal:** Turn the broker's loan book into approved, consent-filtered, compliant retention/marketing campaigns — without ever auto-sending.

**Trigger:** For client-retention or marketing requests (fixed-rate / interest-only / pre-approval expiry outreach, settlement-anniversary or rate-review campaigns, equity-release, stale-lead win-backs, "who should I contact", "build a retention campaign", and follow-ups like "redo the copy" / "refresh the pipeline"), use the `retention-orchestrator` skill.

**Safety:** The harness produces an approved campaign package + send plan and stops there. Sending SMS/email is a separate, explicitly broker-authorised action.
```

## Safety

Sending SMS/email to real clients is outward-facing and irreversible, and a non-compliant message can't be recalled. The harness therefore terminates at an approved package; executing the send is a deliberate, separately-authorised broker action. **This is not legal advice** — the compliance skill is an operational checklist that flags items needing human/legal confirmation.
